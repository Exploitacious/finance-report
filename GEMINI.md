# Finance Report API

## Project Overview
This project is a **FastAPI** web server designed to generate and serve financial reports. It is containerized using **Docker** and deployed via a **GitOps pipeline** involving **GitHub Actions**, **GitHub Container Registry (GHCR)**, and **Watchtower**.

### Main Technologies
- **Language:** Python 3.9
- **Framework:** FastAPI
- **Web Server:** Uvicorn
- **Containerization:** Docker (No Docker Compose)
- **CI/CD:** GitHub Actions (Build & Push to GHCR)
- **Deployment Strategy:** Watchtower (auto-pulling new images)

## Deployment Architecture
- **No Docker Compose:** The `docker-compose.yml` file has been deprecated and removed.
- **GitOps:** Deployment is managed by pushing to the `main` branch, which triggers a GitHub Action to build and push the image to GHCR. Watchtower on the server automatically pulls and restarts the container.
- **Strict Paths:**
  - **Tokens:** Schwab API tokens are strictly read from `/app/tokens/tokens.json`.
  - **Data:** Reports and state files are strictly written to `/app/data`.

## Configuration & Secrets
All secrets and configuration are managed via **Environment Variables** and **Mounted Volumes**.

### 1. Environment Variables (`.env`)
Create a `.env` file (e.g., at `/root/finance-report-data/.env`) containing the following:
```bash
# Schwab Developer API Credentials
SCHWAB_API_KEY=your_app_key_here
SCHWAB_API_SECRET=your_app_secret_here

# FRED (Federal Reserve Economic Data) API
FRED_API_KEY=your_fred_api_key_here

# Optional: News API
NEWS_API_KEY=your_news_api_key_here
```

### 2. Token File (`tokens.json`)
The Schwab API token file (generated by `schwab-py` authentication flow) must be placed in a directory mounted to `/app/tokens`.
- **Host Location:** `/root/finance-report-tokens/tokens.json` (example)
- **Container Location:** `/app/tokens/tokens.json`

### 3. Data Directory
The application writes reports and internal state to `/app/data`.
- **Host Location:** `/root/finance-report-data/` (example)
- **Container Location:** `/app/data`

## Building and Running

### Production Deployment
Run the container with the following command, ensuring all volumes and env files are correctly mapped:
```bash
docker run -d --name finance-report -p 8080:8000 --env-file /root/finance-report-data/.env -v /root/finance-report-data:/app/data -v /root/finance-report-tokens:/app/tokens --restart unless-stopped ghcr.io/<YOUR_GITHUB_USERNAME>/finance-report:latest
```

### Local Development
1. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
2. **Set up local environment:**
   - Create `.env` file with API keys.
   - Ensure you have a valid `tokens.json`.
   - Update `app/sources/schwab.py` or use a mock if you don't have a token file locally (Note: The code now hardcodes `/app/tokens/tokens.json`, so you may need to symlink or adjust for local dev).
3. **Start the server:**
   ```bash
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

## Development Conventions
- **API Entry Point:** `app/main.py`
- **Dependency Management:** `requirements.txt`
- **Error Handling:** The application handles missing Schwab tokens gracefully. It will log a **CRITICAL ERROR** but continue to serve FRED and yfinance data.
- **Automation:** Push to `main` -> GitHub Action -> GHCR -> Watchtower.

## Key Files
- `app/main.py`: The core FastAPI application logic.
- `app/engine.py`: Report generation logic.
- `app/sources/`: Data source implementations (Schwab, FRED, etc.).
- `Dockerfile`: Instructions for building the container image.
- `.github/workflows/deploy.yml`: CI/CD pipeline definition.
